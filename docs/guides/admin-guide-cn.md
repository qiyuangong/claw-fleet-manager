## 0. 界面总览

在浏览器中打开 Claw Fleet Manager 后，您将看到三个区域。

![仪表盘总览：左侧为侧边栏，中间为主面板，主面板顶部为标签栏](screenshots/00-dashboard.png)

**侧边栏（左列）**

| 元素 | 功能 |
|------|------|
| 实例列表 | 每个实例对应一个按钮，点击即可打开（仅非管理员用户可见） |
| Dashboard | 全舰队会话总览：状态分布、活动看板、筛选器（仅管理员） |
| Manage Instances | 创建、重命名或删除实例（仅管理员） |
| 运行中 | 实时监控全舰队当前活跃会话（仅管理员） |
| 活动 | 包含筛选、排序和搜索的历史会话表格（仅管理员） |
| Users | 创建和管理用户账号（仅管理员） |
| Fleet Config | 全局舰队设置（仅管理员） |

**主面板（中间）**

显示所选实例或管理面板的详细信息。

**标签栏（主面板顶部）**

选中实例后，标签栏会随运行时而变化：

- OpenClaw 实例通常会显示：Overview · Activity · Logs · Config · Metrics · Control UI · Feishu · Plugins
- Hermes Docker 实例保留通用标签，如 Overview · Logs · Config · Metrics，并隐藏 OpenClaw 专属工具

> **注意：** 非管理员用户只能看到分配给自己的实例，无法看到管理员导航项。

## 1. 创建新实例

当您需要向舰队中添加新的 OpenClaw 或 Hermes 受管实例时，使用此功能。

**步骤：**

1. 在侧边栏中，点击 **Manage Instances**（位于 Admin 区域下）。

   ![侧边栏中高亮显示 Manage Instances 按钮](screenshots/01-sidebar-manage-instances.png)

2. 点击 **Add Instance**。

   ![实例管理面板中的 Add Instance 按钮](screenshots/01-add-instance-button.png)

3. 在弹出的下拉菜单中选择要创建的实例类型：

   - **Create OpenClaw Docker**
   - **Create OpenClaw Profile**
   - **Create Hermes Docker**

   > **运行时说明：** Hermes 仅支持 Docker，不提供 Hermes profile 选项。

4. 在打开的对话框中，输入实例名称。

   ![显示名称输入框的 Add Instance 对话框](screenshots/01-add-instance-dialog.png)

   > **命名规则：** 只能使用小写字母、数字和连字符（例如 `team-a`、`dev-1`）。名称 `main` 为保留字，请勿使用。

5. 如果您选择的是 **Create OpenClaw Profile**，可按需填写 **Gateway Port**；留空则由系统自动分配。

6. 如果您选择的是 Docker 方案，且需要按实例覆盖镜像 / CPU / 内存 / 端口步长，可展开 **Advanced Docker Config** 进行设置。

7. 点击与所选类型对应的创建按钮。

8. 新实例会出现在实例管理表中。点击 **Open** 即可检查详情。

> **创建后：** 只要实例自举成功，管理器就会自动启动它。后续可在[第 2 节](#2-启动--停止--重启实例)中执行停止、重启或状态确认。

## 2. 启动 / 停止 / 重启实例

使用此功能控制实例的运行状态。

**步骤：**

1. 在侧边栏中点击 **Manage Instances**，然后在目标实例所在行点击 **Open**。

2. 确认当前处于 **Overview** 标签页（默认已选中）。

   ![Overview 标签页，显示状态徽章及 Start / Stop / Restart 按钮](screenshots/02-overview-tab.png)

3. 点击所需操作：

   | 按钮 | 使用场景 | 可用状态 |
   |------|----------|----------|
   | **Start** | 启动已停止的实例 | 实例已停止 |
   | **Stop** | 关闭正在运行的实例 | 实例正在运行 |
   | **Restart** | 停止后立即重新启动 | 实例正在运行 |

4. 面板右上角的**状态徽章**将更新为 `running` 或 `stopped`。

> **提示：** 编辑实例配置后（见第 10 节），请使用 **Restart** 使更改生效。

## 3. 重命名实例

使用此功能为实例指定新名称。重命名仅适用于**已停止**的实例。

**步骤：**

1. 在侧边栏中点击 **Manage Instances**（位于 Admin 区域下）。

2. 找到需要重命名的实例。如果实例正在运行，请先点击同行的 **Stop** 按钮将其停止。

3. 点击该实例操作行中的 **Rename**。

   > 实例运行时 **Rename 按钮不可用**，按钮提示会说明原因。

4. 在弹出的对话框中输入新名称。

   > **命名规则：** 只能使用小写字母、数字和连字符（例如 `team-a`、`worker-2`）。Profile 实例和 Docker 实例均适用此规则。

5. 点击 **Rename**。实例将以新名称重新出现。

> **重命名后：** 引用旧名称的用户实例分配会自动更新。在 Overview 标签页中点击 **启动** 将实例恢复运行。

## 3b. 迁移 OpenClaw 实例

使用此功能可在 **Profile** 与 **Docker** 模式之间迁移 OpenClaw 实例，并保留同一个受管实例 ID。Hermes 实例不会显示此操作。

**步骤：**

1. 在侧边栏中点击 **Manage Instances**，然后在要迁移的实例所在行点击 **Open**。保持停留在 **Overview** 标签页。

2. 点击 **Migrate**。

3. 选择目标模式：

   - **Docker**：将 profile 模式的 OpenClaw 实例迁移到受管容器
   - **Profile**：将 Docker 模式的 OpenClaw 实例迁移到原生 `openclaw --profile` 进程

4. 保持 **Remove source instance after migration** 为启用状态。当前服务端要求在迁移过程中移除源实例，以避免重复实例 ID。

5. 点击 **Migrate**。

6. 迁移完成后，如有需要刷新实例面板，并确认 **Type** 字段已显示新的模式。

## 4. 用户管理

使用此功能创建账号、控制用户可访问的实例，以及重置密码。

### 4a. 打开用户管理

点击侧边栏中的 **Users**（位于 Admin 区域下）。

![侧边栏中高亮显示 Users 按钮](screenshots/03-sidebar-users.png)

Users 面板将列出所有账号。

![用户管理面板，显示用户列表表格](screenshots/03-users-panel.png)

---

### 4b. 创建用户

1. 在 **Add User** 区域中填写新用户信息。

   ![显示用户名和密码输入框的 Add User 对话框](screenshots/03-add-user-dialog.png)

2. 输入**用户名**和**初始密码**。

3. 设置**角色**：
   - **Admin** — 可访问所有实例和管理面板
   - **User** — 只能访问分配给该用户的实例

4. 点击 **Add**。

---

### 4c. 为用户分配实例

拥有 **User** 角色的用户只能访问其分配列表中的实例。

1. 在表格中找到该用户，点击 Actions 列中的 **Instances** 按钮（仅对非管理员用户显示）。
2. 勾选或取消勾选该用户可访问的实例。
3. 点击 **Save**。

> **重要：** 每个实例同一时间只能属于一个用户。将实例分配给当前用户时，会自动从上一位拥有者的访问列表中移除。

![用户管理面板](screenshots/03-users-panel.png)

---

### 4d. 重置密码

1. 在表格中找到该用户，点击 **Reset Password**。
2. 输入新密码并确认。
3. 点击 **Save**。

> **注意：** 用户可在 My Account 面板中自行修改密码。

## 5. 审批设备

当用户的浏览器或客户端正在等待批准以连接到实例的 Control UI 时，使用此功能。

> **适用范围：** 暴露了 Control UI 的 OpenClaw 实例。

**步骤：**

1. 在侧边栏中点击 **Manage Instances**，然后在目标实例所在行点击 **Open**。

2. 点击 **Control UI** 标签页。

3. 如有待审批设备，将显示一个黄色卡片，列出设备数量、每台设备的 IP 地址和请求 ID。

   ![Control UI 标签页，显示黄色待审批设备卡片及 Approve 和 Approve All 按钮](screenshots/04-controlui-pending.png)

4. 点击特定设备旁的 **Approve** 单独审批，或点击 **Approve All** 一次性审批所有设备。

5. 已审批的设备将立即从列表中消失。

> **没有待审批设备？** 如果卡片未出现，表示当前没有设备等待审批。

## 6. 飞书配对

使用此功能将实例连接到飞书（Lark）机器人频道，并审批用户的配对请求。

> **适用范围：** OpenClaw 实例。

### 6a. 配置飞书凭据

每个实例只需配置一次（或在凭据更改时重新配置）。

1. 在侧边栏中点击 **Manage Instances**，然后在目标实例所在行点击 **Open**。进入实例面板后，点击 **Feishu** 标签页。

   ![Feishu 标签页，显示 App ID、App Secret、Group Policy 和 Save Config 按钮](screenshots/05-feishu-config.png)

2. 输入飞书开发者控制台中的 **App ID** 和 **App Secret**（例如 `cli_xxx` 及对应的 secret）。

3. 设置 **Group Policy**（群组策略）：
   - **Open** — 机器人加入的任何群组均可使用
   - **Allowlist** — 仅限审批过的群组
   - **Disabled** — 禁止群组使用机器人

4. 勾选或取消勾选 **Require Mention** — 勾选后，用户必须 @提及机器人才能获得回复。

5. 点击 **Save Config**。

6. 前往 **Overview** 标签页，点击 **Restart** 使凭据生效。

---

### 6b. 审批飞书配对请求

当飞书用户向机器人发送配对命令后，其配对码将显示在此处。

1. 先在 **Manage Instances** 中点击目标实例行里的 **Open**，再点击 **Feishu** 标签页。

2. 在 **Pending Pairing Requests** 区域中找到该用户的配对码。

   ![Feishu 标签页，显示带有 Approve 按钮的待配对请求卡片](screenshots/05-feishu-pending.png)

3. 点击配对码旁的 **Approve**。

> **没有待处理请求？** 该区域将显示"No pending pairing requests."——用户可能尚未发送命令，或机器人未运行（请确认实例已启动且飞书凭据已保存）。

## 7. 查看日志与监控运行状态

### 7a. 实时日志流

使用此功能实时观察实例的运行情况，或排查问题。

1. 先在 **Manage Instances** 中点击目标实例行里的 **Open**，再点击 **Logs** 标签页。

   ![Logs 标签页，在终端风格面板中显示流式日志输出](screenshots/06-logs-tab.png)

2. 日志自动流式输出，向上滚动可查看历史记录。

> **提示：** 如果实例已停止，日志面板将显示关闭前最后捕获的输出。

---

### 7b. CPU 与内存指标

使用此功能检查实例是否处于高负载或内存不足的状态。

1. 先在 **Manage Instances** 中点击目标实例行里的 **Open**，再点击 **Metrics** 标签页。

   ![Metrics 标签页，显示 CPU 和内存使用量的时序图表](screenshots/06-metrics-tab.png)

2. 图表每隔几秒实时更新。

   | 图表 | 显示内容 |
   |------|----------|
   | CPU | 已分配 CPU 的使用百分比 |
   | Memory | 已使用内存与可用总内存 |

## 8. 查看实例活动

使用此功能查看特定实例的会话历史——包括已运行的会话、状态、Token 用量和费用。

> **适用范围：** 支持 session 的 OpenClaw 实例。Hermes 实例不会显示 **Activity** 标签页。

**步骤：**

1. 打开实例面板——在侧边栏点击 **Manage Instances**，然后点击目标实例行中的 **Open**。实例面板打开后，点击 **Activity** 标签页。

2. 标签页显示会话列表，包含以下信息：
   - 会话标题和 key
   - 状态（`running`、`done`、`failed`、`killed`、`timeout`）
   - 使用的模型
   - Token 用量和预估费用
   - 相对时间戳

3. 使用状态筛选器（全部 · 活跃 · 已完成 · 错误）和时间筛选器（今天 · 24 小时 · 7 天 · 全部）缩小列表范围。

4. 使用工具栏中的视图切换在**看板视图**和**表格视图**之间切换。

> **提示：** 看板视图按状态列（运行中、已完成、失败、被终止/超时）分组显示会话。表格视图支持按 Token 数量、费用或最后更新时间排序。

## 9. 安装或卸载插件

使用此功能为实例添加或移除扩展。

> **适用范围：** 暴露了插件能力的实例；目前即 OpenClaw 实例。

### 9a. 安装插件

1. 先在 **Manage Instances** 中点击目标实例行里的 **Open**，再点击 **Plugins** 标签页。

   ![Plugins 标签页，显示已安装插件列表和 Install Plugin 按钮](screenshots/07-plugins-tab.png)

2. 点击 **Install Plugin**。

3. 输入插件标识符（例如 `@anthropic/plugin-name`）。

4. 点击 **Install**。安装完成后，插件将出现在已安装列表中。

---

### 9b. 卸载插件

1. 在已安装列表中找到该插件。

2. 点击旁边的 **Remove**（或垃圾桶图标）。

3. 在弹出的确认提示中确认移除。

> **注意：** 部分插件在安装或卸载后可能需要重启实例。请使用 **Overview** 标签页进行重启。

---

## 10. 编辑实例配置

使用此功能修改实例的设置——模型、API 密钥、提供商或其他任何 `openclaw.json` 字段。

**步骤：**

1. 先在 **Manage Instances** 中点击目标实例行里的 **Open**，再点击 **Config** 标签页。

   ![Config 标签页，Monaco JSON 编辑器显示 openclaw.json 内容](screenshots/08-config-tab.png)

2. 编辑器显示该实例当前的 `openclaw.json`，修改需要更改的字段。

3. 点击 **Save**。

   > **JSON 错误：** 编辑器会以红色高亮标注语法错误，请在保存前修正——无效的 JSON 将被拒绝。

4. 前往 **Overview** 标签页，点击 **Restart**。配置更改仅在重启后生效。

   > **重要：** 请勿跳过重启步骤——实例在重启前将继续使用旧配置运行。

## 11. 实时监控运行中的会话

使用此功能实时查看全舰队正在发生的活动。

**步骤：**

1. 在侧边栏中点击 **运行中**（位于 Admin 区域下）。

2. 点击 **启动** 开始实时轮询。面板每 300 毫秒刷新一次。

3. 面板为每个活跃会话显示一张卡片，包含：
   - 实例 ID
   - 会话标题和 key
   - 使用的模型
   - 当前 Token 数量和费用
   - 最近消息的滚动预览

4. 使用搜索框按实例 ID、会话 key、模型或消息文本筛选。

5. 点击实例 ID 链接可直接跳转到该实例面板。

6. 点击 **停止** 暂停实时轮询。

> **注意：** 监控状态会通过浏览器 localStorage 在页面刷新后保持。

## 12. 查看历史会话

使用此功能查找全舰队中已完成、失败或被终止的会话。

**步骤：**

1. 在侧边栏中点击 **活动**（位于 Admin 区域下）。

2. 面板显示来自所有实例的可筛选、可排序会话表格。

3. 使用状态标签筛选：**全部 · 活跃 · 已完成 · 错误**。

4. 按时间窗口筛选：**今天 · 最近 24 小时 · 最近 7 天 · 全部**。

5. 点击列标题按该列排序。

6. 使用搜索框通过标题、key、模型或最近消息预览查找会话。

7. 使用视图切换在**看板视图**（看板样式列）和**表格视图**之间切换。

8. 点击实例 ID 可跳转到该实例面板。

## 13. 管理员仪表盘

使用此功能获取全舰队会话健康状况和活动趋势的总览。

**步骤：**

1. 在侧边栏中点击 **Dashboard**（位于 Admin 区域下）。

2. 仪表盘包含以下面板：

   | 面板 | 显示内容 |
   |---|---|
   | 指标网格 | 总会话数、活跃实例数、当前运行中（可点击）、平均 CPU、总内存 |
   | 状态分布 | 运行中 / 已完成 / 失败 / 被终止/超时的比例堆叠条——点击某段可筛选 |
   | 吞吐量 | 每个时间分组的会话完成数直方图；可切换最近 24 小时和最近 7 天 |
   | 运行时分布 | 按会话时长的分布（运行中、< 5 分钟、5–30 分钟、30 分钟–2 小时、2 小时以上） |
   | 热点实例 | 按会话数排名的实例列表 |
   | 模型负载 | 按 Token 用量排名的模型列表 |

3. 使用仪表盘上方的**搜索框**和**状态 / 时间筛选器**来缩小各面板统计所用的会话范围。

4. 点击**状态分布**条中的某个分段，或点击**当前运行中**指标卡，可将所有面板聚焦于该状态。

5. 点击 **Reset filters** 恢复默认视图。

6. 点击 **Refresh** 手动获取最新数据。
